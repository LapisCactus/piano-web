# Webブラウザピアノアプリ

スマホでもブラウザから利用できるピアノアプリの設計

## 基本方針

ブラウザだけで完結し、低遅延かつ関数生成にこだわる。
以下の構成とする。

```
タッチイベント
      │
      ▼
SharedArrayBuffer
      │
      ▼
AudioWorklet
      │
      ├─ 鍵盤状態
      ├─ ADSRエンベロープ
      ├─ Karplus-Strongまたは簡易物理モデル
      ├─ 共鳴フィルタ
      └─ 出力
```

この方式では、音声生成は常時リアルタイムで進み、鍵盤の押下状態だけを高速に反映します。JavaScriptメインスレッドとオーディオスレッドの通信には `SharedArrayBuffer` と `Atomics` を使うことで、メッセージキューによる余分な遅延を抑えられます。

この構成なら、ブラウザだけで動作しながら、体感上かなり自然な演奏が可能なレベルまで遅延を小さくできます。ピアノのような打鍵楽器では、音の生成開始を待つのではなく、「オーディオ処理は常時走らせておき、押鍵情報だけを即座に反映する」という設計が最も効果的です。

## 設計

ファイル構成は次のようにします。

```
/
├── index.html
├── style.css
├── app.js
└── synth-processor.js   ← AudioWorklet
```

特徴

* 2オクターブ（C4～B5）
* 画面幅100%
* 高さ300px
* マウス・タッチ両対応
* AudioWorkletで常時音声生成
* メインスレッドは「どの鍵が押されているか」だけ通知
* 音色はサンプルを使わず関数生成
* ADSR付き
* 将来的にKarplus-Strongへ差し替えやすい構造

## 音源

最初はこんな感じの合成にします。

```
fundamental
+ 0.5*sin(2f)
+ 0.25*sin(3f)
+ hammer noise
↓
low-pass
↓
ADSR
```

これだけでも電子ピアノ程度には近づきます。

## UI

```
┌───────────────────────────────────┐
│  vol ██--                         │
│ ┌──────────────────────────────┐  │
│ │  █   █ █   █ █ █   █ █   █ █ │  │
│ │  █   █ █   █ █ █   █ █   █ █ │  │
│ │ A B C D E F G A B C D E F G A│  │
│ └──────────────────────────────┘  │
└───────────────────────────────────┘
```

白鍵15本、黒鍵10本を実際のピアノ比率で配置します。

押された鍵は

* 白→薄青
* 黒→水色

に光ります。

ボリューム調整のスライドで音を小さくできます。

## 実装内容

* 2オクターブ（C4～B5）の鍵盤
* 画面幅100%、高さ約300px
* ピアノ配列の白鍵・黒鍵
* スマホの**マルチタッチ対応**（`pointerdown` / `pointermove` / `pointerup` と `setPointerCapture` を使用）
* 指をスライドすると別の鍵へ移動して演奏
* 複数の指で和音演奏
* PCではマウス操作
* AudioWorkletで常時音声生成
* メインスレッドは押下状態のみ通知
* 関数合成によるピアノ風音色（ADSR付き）
* 将来Karplus-Strongへ置き換えやすい設計

さらに、スマホ向けに以下も対応します。

* 画面スクロール防止（`touch-action: none`）
* 長押しメニュー無効
* ダブルタップズーム防止
* 横向き・縦向きどちらでも演奏可能
* iPhoneのオーディオ開始制約（最初のタップで `AudioContext.resume()`）
